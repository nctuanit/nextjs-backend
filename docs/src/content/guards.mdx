# Guards (Bảo vệ đường dẫn)

Một bộ bảo vệ (guard) là một lớp gắn kèm với decorator `@Injectable()`, bắt buộc phải tuân theo hệ thống triển khai (implement) từ interface `CanActivate`.

Guards chỉ có một trách nhiệm duy nhất. Chúng quyết định quyền đi tiếp của một request sẽ tiến đến Route Handler (Hàm xử lý phương thức Controller), theo từng quy trình bảo trợ (authorization/authentication) nhất định theo thời gian chạy (Ví dụ như thiết lập điều kiện phân tầng, quyền quản trị người dùng cơ sở hoặc mã xác thực bảo vệ API).

## Authentication Guard (Phân loại Quyền Cơ Bản)

Các giải pháp truyền thống trong ExpressJS tạo ra thói quen cài cắm các phần mềm ứng dụng xác thực chuỗi token bằng quy chuẩn middleware tự hành qua biến `next()`. Tuy nhiên, trình tiện ích này không thể khai thác trọn hệ tham số để nhận định hành tung rành mạch từ cấp endpoint phân luồng tuyến đường nào được cài trước hàm `next()`.

Guards tiên tiến hơn so với middleware ở đặc trưng tận dụng mô hình thiết lập ngữ cảnh (`Context`), có khả năng nắm bắt hệ thống chức năng nào của một nhóm controller được khởi chạy tại quá trình hoạt động tuyến tính.

```typescript
import { Injectable, CanActivate, Context } from './src';

@Injectable()
export class AuthGuard implements CanActivate {
  async canActivate(context: Context): Promise<boolean> {
    const request = context.request;
    const token = request.headers.get('authorization');
    
    // Khuyến nghị: triển khai kỹ thuật đánh giá Token JWT
    const isValid = validateToken(token);
    
    // Nếu kết quả trả về false, hệ thống báo lỗi không khởi tạo luồng
    return isValid;
  }
}
```

Nếu đánh giá bảo mật thông báo trình chặn trạng thái boolean là `false`, library sẽ dừng luồng truy xuất request tại bộ định vị và phản sinh mã lỗi HTTP Exception `403 Forbidden`.

## Áp Đụng Cấu Theo Guards (Binding Guards)

Sử dụng nguyên tắc đồng dạng khi ứng dụng cho bộ kiểm tra cơ chế Pipes, lệnh trói buộc (binding) phần tử khai báo bằng cú pháp `@UseGuards()`. Định nghĩa hoàn toàn có thể cài được tự nhiên theo 2 cấp: Đánh vào tham gia Route Method (Khóa riêng rẽ 1 hàm) hoặc theo Controller Level (Bọc phủ bảo vệ cho cả phân lớp).

```typescript
import { Controller, Get, UseGuards } from './src';
import { AuthGuard } from './auth.guard';

@Controller('/cats')
// Buộc toàn bộ tập tuyến đường API của Controller thi hành thủ tục Authentication 
@UseGuards(AuthGuard)
export class CatsController {
  
  @Get()
  findAll() {
    return 'Dữ liệu được xác thực cẩn mật';
  }
}
```

Có những môi trường yêu cầu trình bảo vệ nghiêm ngặt? Không gặp phải bất kỳ trở ngại nào. Lập trình viên có thể mở rộng danh sách tập các Guards ngăn bằng dấu phẩy truyền thống:

```typescript
@UseGuards(AuthGuard, BannedUserGuard, RolesGuard)
```
