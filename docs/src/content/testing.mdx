# Testing Utilities

`Test.createTestingModule()` provides isolated unit and integration testing, inspired by NestJS.

## Unit Testing

Create a testing module with providers and resolve them:

```typescript
import { Test } from 'next-js-backend';
import { UserService } from './user.service';

const module = await Test.createTestingModule({
  providers: [UserService],
}).compile();

const service = await module.get(UserService);
expect(service.findAll()).toEqual([...]);
```

## Mocking Providers

Override providers with mock values using `overrideProvider()`:

```typescript
const mockService = {
  findAll: () => [{ id: 1, name: 'Mock' }],
  create: (name: string) => ({ id: 2, name }),
};

const module = await Test.createTestingModule({
  controllers: [UserController],
  providers: [UserService],
})
  .overrideProvider(UserService).useValue(mockService)
  .compile();

const service = await module.get(UserService);
expect(service.findAll()).toEqual([{ id: 1, name: 'Mock' }]);
```

## Integration Testing

Use `createApp()` to build a real Elysia app for HTTP-level testing:

```typescript
const module = await Test.createTestingModule({
  controllers: [UserController],
  providers: [UserService],
}).compile();

const app = await module.createApp();

const response = await app.handle(
  new Request('http://localhost/users')
);

expect(response.status).toBe(200);
const data = await response.json();
expect(Array.isArray(data)).toBe(true);
```

## Override Strategies

### useValue

Replace a provider with any value:

```typescript
.overrideProvider(ConfigService).useValue({
  get: (key) => 'mock-value',
})
```

### useClass

Replace with a different class implementation:

```typescript
.overrideProvider(UserService).useClass(MockUserService)
```
