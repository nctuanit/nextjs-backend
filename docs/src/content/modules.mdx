# Khối Chức Năng (Modules)

Một **module** là một lớp (class) có khai báo decorator `@Module()` ở cấp độ tiền xử lý. Theo quy ước, `@Module()` decorator cung cấp siêu dữ liệu (metadata) mà library dựa vào để xây dựng định hình của tổ chức dữ liệu ứng dụng.

Mọi ứng dụng tiêu chuẩn đều có ít nhất một module gốc, gọi là **Root Module** (Mô-đun gốc phân rễ). Từ căn cứ này, library khởi đầu để tìm cấu trúc biểu đồ Application Graph - hệ thống dữ liệu chịu trách nhiệm xử lý các phần mở rộng chức năng cài cắm theo kiểu phân bổ tham số (dependency injection).

Dù các giải pháp khởi đầu với tỷ lệ vừa và nhỏ quy ước là chỉ cần module Root Module là hoàn toàn hợp quy chuẩn. Nhưng chúng tôi hướng dẫn: Cấu trúc chia nhóm thành từng lớp mô đun độc lập sẽ tổ chức ranh giới mã nguồn sắc bén giúp các hạng mục tách bạch theo tính năng logic.

## Cấu Trúc Trình Cắm `@Module()` 

Định nghĩa `@Module()` tiếp nhận duy nhất một biến cấu hình kiểu đối tượng:

| THUỘC TÍNH (PROPERTY) | KHÁI NIỆM TRỌNG ĐIỂM |
| --- | --- |
| `controllers` | Một mảng đối tượng nhận vào những lớp Controller chỉ định để thêm chúng vào sơ đồ gốc ứng dụng. |
| `providers` | Các dịch vụ cài cắm, trình hỗ trợ, factory có thể đóng vai trò khởi chạy cung cấp qua IoC container. |
| `imports` | Một mảng danh sách trỏ tính năng của những cấu hình Module khác (Ví dụ khai báo `CatsModule`,  nhằm đưa nó vào quá trình khởi tạo ứng dụng). |

## Cơ Chế Ghi Nhận Nội Bộ 

Ở phiên bản cơ sở tĩnh hiện tại, phiên bản `ElysiaFactory` của library đang làm việc qua một cơ chế phân vùng Global Scope (Khu vực hệ thống tổng thể dùng chung). Bằng việc gửi định tuyến ứng dụng ở điểm Root Module tại hàm khởi chạy `ElysiaFactory.create(AppModule)`, trình quản lý sẽ thu gom các định nghĩa của `controllers` và `providers` nhằm tạo tập hợp dữ kiện lớn hoạt động đồng bộ.

```typescript
import { Module } from './src';
import { CatsController } from './cats.controller';
import { CatsService } from './cats.service';
import { AppController } from './app.controller';

@Module({
  controllers: [AppController, CatsController],
  providers: [CatsService],
})
export class AppModule {}
```

**Khuyến cáo hệ thống**: So với cấp độ NestJS thuần thục thường vận dụng tính năng phân tầng thừa kế cấp tiến của tham số `imports`, việc xử lý đối chiếu với engine `ElysiaFactory` của library đang đánh trọng tâm theo kỹ thuật cấu hình biên dịch nhị phân hệ thống ở dạng dẹt (flat hierarchy). Bởi thế, cấu trúc toàn bộ tính năng ở cấp độ lớp cấu hình vỏ cao nhất ROOT MODULE (thông thường là `AppModule`) giúp cho tiến trình Dependency Injection dò bản quyền dữ liệu hoạt động trơn tru ổn định trong điều kiện engine chưa khởi động đệ quy nhiều chiều!
