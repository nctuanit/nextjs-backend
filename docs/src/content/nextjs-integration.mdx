# Tích Hợp Vào Route Next.js API

Điểm mạnh nhất của **Next.js-Backend** là tính kế thừa nguyên bản sức mạnh của **ElysiaJS**, nền tảng tương thích tuyệt đối với chuẩn Web Standard và WinterCG. Điều này có nghĩa là mọi phương thức xử lý của framework đều tiếp nhận chuẩn `Request` và trả về chuẩn `Response` tiêu chuẩn.

Nhờ lợi thế này, bạn có thể **nhúng thẳng toàn bộ ứng dụng Backend vào duy nhất một Route Catch-All của Next.js (App Router)** thay vì phải chạy máy chủ Bun trung gian. Framework sẽ trở thành một module nhận xử lý trực tiếp tất cả các yêu cầu từ cấu trúc giao diện của Next.js!

## Cấu Hình Tích Hợp Trên App Router

Bạn chỉ cần tạo một định tuyến bao trùm luồng API (catch-all route) tại khu vực thư mục `app/api/[[...slug]]/route.ts` của ứng dụng Next.js. Bản đồ thư mục của dự án sẽ có hình hài như sau:

```text
my-next-app/
├── app/
│   └── api/
│       └── [[...slug]]/
│           └── route.ts     <-- Nơi cầu nối Next.js x Elysia tụ hội
├── server/
│   ├── app.module.ts        <-- Code Backend (Controllers, Services)
│   ├── users.controller.ts
│   └── users.service.ts
└── next.config.mjs
```

### Triển Khai Gọi `createNextJsHandlers`

Để đơn giản hóa hoàn toàn quy trình này, framework cung cấp sẵn một tiện ích sinh mã `createNextJsHandlers`. Bạn không cần phải khởi tạo các biến Singleton phức tạp. Chỉ cần truyền `AppModule` vào, framework sẽ tự động cung cấp lại toàn bộ các phương thức định tuyến xử lý gốc (GET, POST, PUT, DELETE,...) mà Next.js App Router yêu cầu phải export.

Dưới đây là nội dung cài đặt cực kỳ tinh gọn của tệp `app/api/[[...slug]]/route.ts`:

```typescript
// app/api/[[...slug]]/route.ts
import { ElysiaFactory } from 'next-js-backend'; // Thay bằng tên package của bạn
import { AppModule } from '@/server/app.module';

// 1. NGĂN CHẶN CACHE TĨNH (STATIC CACHING): 
// Bắt Next.js luôn chạy API động thay vì lưu lại nội dung từ lần đầu gọi
export const dynamic = 'force-dynamic';

// 2. TÍCH HỢP TẤT CẢ PHƯƠNG THỨC:
// Lệnh gọi `globalPrefix` giúp ánh xạ các endpoint (ví dụ `@Controller('/users')`) 
// thành `/api/users` để khớp với vị trí đặt file route.ts của Next.js
export const { GET, POST, PUT, PATCH, DELETE, OPTIONS } = ElysiaFactory.createNextJsHandlers(AppModule, { 
    globalPrefix: '/api' 
});
```

## Ưu Điểm Đột Phá Của Việc Tích Hợp Trực Tiếp

1. **Hiệu suất tuyệt đối**: Phương pháp trên loại bỏ hoàn toàn độ trễ do mạng lưới Proxy hoặc các cổng kết nối ngoại tuyến gỡ chặn gây ra. Next.js đóng vai trò máy tiếp đón và đẩy thẳng vào tầng lõi Backend.
2. **Loại bỏ hệ quản trị song song**: Không cần phải thao tác thiết lập tính năng đa luồng như `concurrently`. Toàn bộ dữ liệu hệ thống chạy chung trên một lệnh khởi động ứng dụng Next.js duy nhất.
3. **Triển khai máy chủ cực mượt**: Các nền tảng triển khai như **Vercel** hay **Node.js** nhận diện và đóng gói chính xác hệ nguồn mà không yêu cầu cài đặt phần mềm Docker. Cơ sở ứng dụng vận hành xuyên suốt ngay bên trong ranh giới hàm Serverless.
4. **Không lỗi chia sẻ trạng thái CORS**: Toàn cảnh API phân giải trên nền giao thức `http://localhost:3000/api/*`, đồng bộ toàn phần gốc tên miền cho cơ chế xác thực trình duyệt Cookies / Sessions.

Với hình thức kiến tạo này, bạn đã hội tụ kiến trúc cấp doanh nghiệp (Enterprise Architecture) vào ứng dụng Next.js hoàn hảo, gặt hái 100% tài nguyên của hệ Decorator, Dependency Injection, Guards, PIPES trên không gian Next App Router gốc!
