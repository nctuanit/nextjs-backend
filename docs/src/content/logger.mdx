# Logger Service

Service Logger được cung cấp sẵn bởi `next-js-backend` giúp bạn ghi lại các thông báo hệ thống một cách chuẩn mực, có phân quyền màu sắc, và đo thời gian thực thi chuyên nghiệp (hiển thị `ms`). Nó cũng là nền tảng log mặc định của bản thân Framework (những gì bạn thấy ở Banner lúc boot là từ Service này).

## Cài đặt Cơ bản

Do `Logger` đã được đăng ký ngầm trong lõi, bạn có thể tiêm ảo (`Inject`) nó vào bất cứ Service hay Controller nào thông qua constructor.

```typescript
import { Injectable, Logger } from 'next-js-backend';

@Injectable()
export class PaymentsService {
  // 1. Khai báo Logger kèm tên Ngữ cảnh (Context)
  private readonly logger = new Logger(PaymentsService.name);

  async processPayment(amount: number) {
    this.logger.log(`Bắt đầu xử lý giao dịch: $${amount}...`);
    
    try {
      // Logic...
      this.logger.debug('Đang kết nối tới cổng thanh toán thứ 3');
      
      this.logger.log('Xử lý thành công!');
    } catch (e) {
      // Ghi nhận lỗi có kèm Stack Trace
      this.logger.error('Khách hàng bị từ chối thẻ', e.stack);
    }
  }
}
```

## Các Mức Độ Log (Log Levels)

Framework cung cấp 5 mức độ cảnh báo (Log levels) được format với màu sắc (ANSI Codes) khác nhau trên Terminal:

| Phương Thức | Level | Màu Sắc Giao Diện | Ý Nghĩa Sử Dụng |
| :--- | :--- | :--- | :--- |
| `logger.log()` | LOG | Xanh Lá (Green) | Cột mốc hoạt động bình thường, thông tin cần thiết. |
| `logger.warn()` | WARN | Vàng (Yellow) | Hệ thống gặp trở ngại nhưng chưa bị sập (VD: Thiếu config phụ). |
| `logger.error()` | ERROR | Đỏ (Red) | Lỗi nghiêm trọng, sập tiến trình, hoặc Request thất bại. |
| `logger.debug()` | DEBUG | Tím (Magenta) | Mức độ chi tiết, thường dùng trong lúc code để print biến. |
| `logger.verbose()` | VERBOSE | Xanh Dạ (Cyan) | Thông tin cực kỳ chi tiết, quá trình trung gian. |

> [!NOTE]
> Constructor `new Logger('Context_Name')` chứa tham số thứ nhất để định danh ngầm định cho chuỗi log. Điều này giúp bạn dễ dàng dò file phát ra lỗi đang nằm ở đâu trong đống Terminal Console dày đặc.

## Sử dụng Dạng Trực Tiếp (Static Pattern)

Nếu bạn đang không ở trong class (VD: trong một hàm rời rạc nằm ngoài DI Container), bạn vẫn có thể gọi logger một cách an toàn thông qua hàm Static.

```typescript
import { Logger } from 'next-js-backend';

function myStandaloneFunction() {
  Logger.warn('Hệ thống đang chạy một tiến trình tắt...', 'StandaloneFunc');
}
```
