# Xác Thực JWT (Authentication)

`JwtModule` mang tới một cấu trúc chuyên biệt để xử lý đăng nhập, cấp phát thẻ (Token) và khoá chặn các tuyến đường API nhạy cảm. Được xây dựng trên thư viện lõi `jose`, module này hoạt động trơn tru 100% trên các môi trường khắc nghiệt như **Vercel Edge Runtime** hay Cloudflare Workers mà không bị lỗi tương thích "Crypto Native".

## Thiết Lập Cốt Lõi (JWT Initialization)

Đầu tiên, bạn cần khai báo `JwtModule` ở Module tương ứng. Thường được bọc dưới dạng cấu hình tĩnh dính liền với Secret Key bí mật (thường được import qua biến môi trường).

```typescript
import { Module, JwtModule } from 'next-js-backend';
import { AuthController } from './auth.controller';

@Module({
  imports: [
    JwtModule.register({
      secret: 'MY_SUPER_SECRET_KEY_123', // Nên để ở .env
      expiresIn: '1h', // Hết hạn sau 1 giờ
      alg: 'HS256'
    })
  ],
  controllers: [AuthController]
})
export class AuthModule {}
```

## Phát Hành Token (Sign)

Bên trong Controller đăng nhập, bạn có thể tiêm `JwtService` để tiền hành nhào nặn JSON Token.

```typescript
import { Controller, Post, Body, JwtService } from 'next-js-backend';
import { t } from 'elysia';

@Controller('/auth')
export class AuthController {
  constructor(private jwtService: JwtService) {}

  @Post('/login')
  async login(@Body(t.Object({ username: t.String() })) dto) {
    // 1. Kiểm tra username/password với DB...
    
    const payload = { sub: dto.username, role: 'admin' };
    
    // 2. Cấp mã thông báo
    const token = await this.jwtService.signAsync(payload);
    
    return { accessToken: token };
  }
}
```

## Khoá Đường Đi Ngay Lập Tức (AuthGuard)

Để buộc người dùng phải nộp thẻ Token ở Header khi đi qua HTTP (dưới tham số `Authorization: Bearer <token>`), hệ thống Next.js-Backend đã cấp sẵn một cửa kiểm soát là `AuthGuard`.

Bạn chỉ việc đặt thẻ chặn `@UseGuards(AuthGuard)` bên trên bất cứ Controller hoặc Controller-Method nào bạn muốn. Guard này tự động bóc dỡ mã bí mật và giải nén thông tin của chủ thể dán vào biến `context.user`.

```typescript
// user.controller.ts
import { Controller, Get, UseGuards, AuthGuard, Headers } from 'next-js-backend';

@Controller('/profile')
@UseGuards(AuthGuard) // Toàn bộ route trong này vĩnh viễn bị khoá
export class ProfileController {

  @Get('/me')
  getSecretData(@Headers('authorization') auth: string) {
    // Nếu vào được đây, nghĩa là AuthGuard đã làm tròn bổn phận!
    return { secret: "This is locked content", tokenKey: auth };
  }
}
```

> [!TIP]
> Do Next.js App Router xử lý dữ liệu chuẩn Request, AuthGuard luôn trích xuất chính xác cấu trúc JWT thông qua Header `Bearer` truyền vào. Nếu mã sai, hết hạn hoặc không có mã, người dùng sẽ tự động ăn HTTP Lỗi `401 Unauthorized` có thông báo rõ ràng vào mặt.

## Mã Hoá Mật Khẩu (Password Hashing)

Phân hệ Auth còn cung cấp sẵn `PasswordService` đóng gói động cơ siêu nguyên bản tốc độ cao `Bun.password` hỗ trợ mã hoá `bcrypt` và `argon2id`! 

Bạn có thể tiêm ảo (`Inject`) nó ở khắp mọi nơi để phục vụ đăng ký/đăng nhập:

```typescript
import { Controller, Post, Body, PasswordService } from 'next-js-backend';

@Controller('/users')
export class UsersController {
  constructor(private passwordService: PasswordService) {}

  @Post('/register')
  async register(@Body() dto) {
    // Tự động dùng Bcrypt (Cost mặc định: 10)
    const hash = await this.passwordService.hash(dto.password);
    
    // Hoặc nếu muốn xài Argon2id thế hệ mới bảo mật hơn:
    // const hash2 = await this.passwordService.hash(dto.password, { algorithm: 'argon2id' });

    return { hashedPassword: hash };
  }

  @Post('/verify')
  async login(@Body() dto) {
    const isMatch = await this.passwordService.verify(dto.password, dto.storedHash);
    if (!isMatch) throw new Error("Sai mật khẩu mẹ rồi!");
    return "Đăng nhập thành công";
  }
}
```
