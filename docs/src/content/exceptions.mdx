# Bộ Lọc Ngoại Lệ (Exception Filters)

Library đi kèm với một lớp ngoại lệ (Exceptions Layer) tích hợp sẵn, chịu trách nhiệm xử lý toàn bộ các ngoại lệ không được bắt (unhandled exceptions) trong toàn bộ ứng dụng.

Khi một ngoại lệ xuất hiện mà không được hệ thống bắt (catch) thủ công bởi mã nguồn của bạn, nó sẽ được chuyển đến lớp này. Lớp ngoại lệ sẽ phân tích và tự động gửi một thông báo phản hồi (response) phù hợp và thân thiện đến client.

## Ném Ngoại lệ Tiêu chuẩn (`HttpException`)

Library cung cấp sẵn một lớp `HttpException` lấy từ tệp định nghĩa `@/src`. Đối với các ứng dụng REST hoặc GraphQL API thông dụng, việc gửi một đối tượng phản hồi HTTP tiêu chuẩn khi có điều kiện lỗi xảy ra là một phương pháp tốt (best practice).

```typescript
import { Controller, Get, HttpException } from './src';

@Controller('cats')
export class CatsController {
  @Get()
  async findAll() {
    throw new HttpException('Việc truy cập danh sách này hiện đang bị cấm!', 403);
  }
}
```

Khi client gửi yêu cầu đến endpoint này, phản hồi nhận được sẽ có định dạng như sau:

```json
{
  "statusCode": 403,
  "message": "Việc truy cập danh sách này hiện đang bị cấm!"
}
```

## Xử lý Lỗi Hệ thống Tích hợp

Bằng cách kế thừa cơ chế xử lý lỗi cục bộ thông qua hook `.error()` nguyên bản của Elysia, library tự động thu thập và chuyển đổi định dạng:
- Các đối tượng `Error` tiêu chuẩn của Javascript.
- **Ngoại lệ Xác thực (Validation Exceptions)**: Được tạo ra bởi bộ lọc `ValidationPipe`.

**Ví dụ về trường hợp lỗi không được xử lý cụ thể:**
```typescript
@Get('/crash')
async crashApp() {
   throw new Error("Hệ thống nội bộ gặp sự cố hỏng hóc!");
}
```

Ngoại lệ sẽ được hệ thống tiếp nhận và tự động biên dịch thành luồng thông báo tiêu chuẩn **500 Internal Server Error**, đảm bảo các chi tiết rủi ro nội hàm không bị rò rỉ:

```json
```json
{
  "statusCode": 500,
  "message": "Internal Server Error"
}
```

## Bắt Ngoại Lệ Tuỳ Chỉnh (Custom Exception Filters)

Next.js Backend hỗ trợ mô hình Xử lý ngoại lệ hoàn toàn tương thích với cấu trúc của NestJS, bao gồm việc xây dựng các lớp **ExceptionFilter** kết hợp cùng decorator **`@Catch()`** và **`@UseFilters()`**.

Bạn có thể nắm toàn quyền kiểm soát vòng đời của Response, bắt các `Error` chuyên biệt, ghi log, hoặc định dạng lại cấu trúc lỗi trả về cho client. 

### Bước 1: Tạo Filter Tuỳ Chỉnh
Xây dựng một lớp kế thừa từ `ExceptionFilter` và định nghĩa logic phân giải lỗi bên trong hàm `catch(exception, context)`.

```typescript
// filters/database-exception.filter.ts
import { Catch, ExceptionFilter, Injectable, Logger } from 'next-js-backend';

// Tự định nghĩa một Error tuỳ chỉnh
export class CustomDatabaseError extends Error {
  constructor(message: string) {
    super(message);
    this.name = 'CustomDatabaseError';
  }
}

// Bắt chính xác loại lỗi này, bỏ qua các lỗi hệ thống thông thường
@Injectable()
@Catch(CustomDatabaseError)
export class DatabaseExceptionFilter implements ExceptionFilter {
  private readonly logger = new Logger(DatabaseExceptionFilter.name);

  catch(exception: CustomDatabaseError, context: any) {
    this.logger.error(`Database Exception Caught: ${exception.message}`);
    
    // context là đối tượng Elysia Context, bạn dùng set để đổi Header/Status
    context.set.status = 503; // Service Unavailable
    
    return {
      error: "Lỗi kết nối cơ sở dữ liệu tạm thời.",
      statusCode: 503,
      details: exception.message,
      timestamp: new Date().toISOString()
    };
  }
}
```

### Bước 2: Sử Dụng Filter (Gắn kết)

Sử dụng Decorator `@UseFilters()` để gắn kết Filter của bạn vào từng Router riêng lẻ, hoặc toàn bộ Controller. Library sẽ tự động nhúng Filter của bạn vào Dependency Injection và giải quyết nó:

```typescript
// app.controller.ts
import { Controller, Get, UseFilters } from 'next-js-backend';
import { DatabaseExceptionFilter, CustomDatabaseError } from './filters/database-exception.filter';

@Controller('/users')
export class UsersController {
  
  @Get('/crash')
  @UseFilters(DatabaseExceptionFilter) // Gắn Filter tuỳ chỉnh vào đây
  crashApp() {
     throw new CustomDatabaseError("Connection timeout out after 3000ms");
  }
}
```

> **Ghi chú**: Đừng quên cung cấp class filter của bạn (như `DatabaseExceptionFilter`) vào thẻ `providers` của `AppModule` hoặc Module trực thuộc để Container có thể tiêm các Injectables (ví dụ LoggerService) vào filter!

### Cấu hình Filter Đa Cấp (Global Filters)

Bạn có thể cấu hình fallback hoặc catch-all errors bằng cách khai báo Filter trống `@Catch()` và truyền nó vào luồng cấu hình gốc khi chạy bootstrap:

```typescript
// app.ts
import { ElysiaFactory } from 'next-js-backend';
import { AppModule } from './app.module';
import { GlobalFallbackFilter } from './filters/global.filter';

const bootstrap = async () => {
  const app = await ElysiaFactory.create(AppModule, { 
     globalPrefix: '/api',
     globalFilters: [GlobalFallbackFilter] // Áp dụng Filter cho mọi module
  });
  return app.handle; 
};
```
