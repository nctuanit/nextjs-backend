# DevMode Profiler (Giám Sát Request)

Trong quá trình phát triển, việc quan sát chính xác từng `Request`, `Response Status`, thời gian phản hồi (`Latency`) và tỷ lệ lỗi (`Error Rate`) là cực kì hữu ích. 
Tương tự như NestJs Devtools, **next-js-backend** đã tích hợp sẵn một Module cực nhanh chạy ngầm có tên là **DevModeModule**.

Nó hook thẳng vào tầng `onAfterResponse` của ElysiaJS để đảm bảo rằng tác động tới hiệu năng là con số không (zero-overhead loop), đồng bộ hóa mọi thông tin từ Request mà bạn tạo ra.

## Cài đặt cơ bản

Chỉ cần đăng ký `DevModeModule` trong AppModule của bạn. Bạn nên dùng biến môi trường để chỉ kích hoạt nó ở môi trường Development:

```typescript
import { DevModeModule, ElysiaFactory } from "next-js-backend";

@Module({
  imports: [
    DevModeModule.register({ 
       enabled: process.env.NODE_ENV !== 'production',
       maxHistory: 50 // Mặc định lưu trữ 50 requests gần nhất vào Memory
    })
  ]
})
export class AppModule {}
```

## Các Endpoint Có Sẵn

Ngay khi DevMode được bật, các endpoint tĩnh sau sẽ tự động được expose trên API của bạn:

- **`GET /dev/history`**: Lấy danh sách giới hạn các lịch sử theo thời gian thực (hiển thị body, headers, method, path, v.v.)
- **`GET /dev/stats`**: Báo cáo tổng thể về máy chủ (Tổng lượt request kể từ lúc boot, Tỷ lệ lỗi Error Rate, v.v.)
- **`GET /dev/history/clear`**: Xóa bộ đệm lịch sử thủ công
- **`GET /dev/ping`**: Xác minh đường truyền DevMode đang sống.

## Cơ Chế Hoạt Động (Under the hood)

Không qua Middleware! Do kiến trúc bất đồng bộ (await next()) chặn luồng của Elysia khác biệt với Express, Dev Mode Profiler sử dụng một lifecycle event ngầm tên là `app.onAfterResponse`. Điều này có nghĩa:
1. Module Profiler chạy ở background, ngoài luồng response chính.
2. Không làm tăng `TTFB` (Time to First Byte) của Client.
3. Không làm gián đoạn Controller logic nếu Profiler gặp sự cố.
