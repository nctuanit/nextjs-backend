# Global Middleware

Phần mềm trung gian (Middleware) là tính năng cốt lõi cho phép bạn thực hiện đoạn mã định kỳ mỗi khi có request gửi vào ứng dụng, đồng thời bạn có thể thay đổi các đối tượng `Request` (Req) và `Response` (Res).

Middleware trong cơ chế này cho phép bạn đăng ký và cài đặt toàn cục theo chuẩn NestJS-like cho cả Dependency Injection. Nó hỗ trợ cấu trúc cực mạnh mẽ.

## Định nghĩa Middleware

Một middleware hợp lệ nên implement interface `NestMiddleware`. Bạn phải dán nhãn `@Middleware()` để thông báo rằng lớp này có thể được gắn kết chung.

Chức năng `use` sẽ nhận tham số `req`, `res` và lời gọi móc `next`. Middleware chỉ hoạt động tiếp các Controller kế sau thông qua việc bắt buộc bạn phải gọi hàm `next()`.

```typescript
import { Middleware, NestMiddleware, Request, Response } from 'next-js-backend';

@Middleware()
export class LoggerMiddleware implements NestMiddleware {
  use(req: Request, res: Response, next: () => void) {
    const start = Date.now();
    console.log(`[>>] Req: ${req.method} ${req.url}`);

    // Truyền điều khiển xuống lớp dưới
    next();
  }
}
```

## Áp dụng trên toàn tuyến đường (Global)

Để cài middleware, bạn có thể tiêm MiddlewareConsumer vào một Module nào đó bằng cách triển khai hàm `configure`.

Dùng phương thức `.apply(TênMiddleware).forRoutes()` để gán middleware vào thư mục mà bạn muốn hướng đến. Truyền dấu sao `*` nếu muốn gán trên toàn bộ server.

```typescript
import { Module, MiddlewareConsumer } from 'next-js-backend';
import { LoggerMiddleware } from './logger.middleware';
import { UsersController } from './users.controller';

@Module({
  controllers: [UsersController],
})
export class AppModule {
  configure(consumer: MiddlewareConsumer) {
    consumer
      .apply(LoggerMiddleware)
      // Middleware này sẽ chạy trên mọi endpoint
      .forRoutes('*'); 
  }
}
```

## Chỉ định Route cụ thể (Route level)

Trang bị Middleware cho một vài endpoint nhất định là hoàn toàn khả thi:

```typescript
// Trong hàm configure của class Module
configure(consumer: MiddlewareConsumer) {
  consumer
    .apply(AuthMiddleware)
    // Chỉ bảo vệ các api/v1/users hoặc endpoint được định tuyến
    .forRoutes({ path: 'users', method: 'GET' }, { path: 'posts', method: 'POST' });
}
```

Nó hỗ trợ tính năng định tuyến cho từng Pattern Path. Các Controllers và Action vẫn nhận được thông tin từ `Request` mà Middleware xử lý trước đó.
